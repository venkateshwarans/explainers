The Agent Stack Bet

Production AI agents keep breaking — not because the prompts are wrong, but because the infrastructure underneath was never built for them. Here's the four-layer fix.

TL;DR — Most production agents today run on duct-taped infrastructure: shared credentials, siloed context, session-bound execution, and custom plumbing that teams rebuild from scratch each time. Addy Osmani argues teams have ~12 months to make four architectural bets — on identity, context, persistence, and platform — or get stuck at "sophisticated demo" forever.

Why agents keep failing in production

When a production agent misbehaves, the instinct is to improve the prompt. But the real culprits are usually invisible: shared service accounts with no audit trail, context fragmented across a dozen systems the agent can't simultaneously see, and state that vanishes the moment a session token expires.

Addy Osmani calls this governance debt — security and reliability risk that accumulates silently, not through bad decisions but through the absence of the right infrastructure. It's a stack failure, not a people failure.

The four bets

Each bet is a shift from something that works in demos to something that works in production. Together they form a coherent architecture — you can't fully solve persistence without identity, and universal context is hollow without platform-level delivery.

Bet 1 · Agent Identity Bet #1

Today: Agents are "ghosts" — they borrow service accounts or inherit human OAuth tokens. Every action looks like it came from infra-prod-001, alongside four other agents and three humans. When something goes wrong, attribution is impossible.

The bet: Move identity from the application layer to the platform layer. Agents need unforgeable machine certificates, recognized at the network level, with permissions enforced at the source. A connection to a restricted system either opens or it doesn't — no middleware gate that can be bypassed with a cleverly worded prompt.

  • Actions become attributable: every database write, every API call, every credential use logs the specific agent
  • Permissions become revocable: pull the certificate, the agent stops — no hunting down shared passwords
  • Security becomes auditable: compliance teams can answer "what did our agents access last Tuesday?"
Bet 2 · Universal Context Bet #2

Today: Teams burn engineering hours on "undifferentiated plumbing" — custom serialization, bespoke session stores, hand-rolled memory layers. The result: a browser agent sees one tab, a desktop wrapper sees dragged files, but neither can simultaneously reason over CRM records, ERP approvals, data warehouse trends, and ticketing history.

The bet: Context integration at the platform level, not the application level. Agents should have a unified view across all business systems — not because each system was painstakingly connected by hand, but because the platform provides a shared semantic layer.

  • Without this: agents are expensive autocomplete for one system at a time
  • With this: agents can reason about cross-system causality ("contract terms in ERP explain the anomaly in AP invoices")
  • The ceiling for siloed agents is "slightly better spreadsheet autocomplete" — Osmani's bluntest line
Bet 3 · Persistence Across Disconnects Bet #3

Today: Most agents hit hard ceilings — time-based, token-based, or governance-based. When a session ends, work disappears. Humans reassemble state from transcripts. For an agent running a procurement workflow that spans multiple vendor approvals across three days, this is fatal.

The bet: State and checkpointing that survives restarts, disconnects, redeploys, and model version changes. Long-horizon memory with automatic summarization so context doesn't exhaust token limits over multi-week tasks. And critically: first-class human-in-the-loop primitives — the ability to pause and ask permission rather than silently deciding authority.

  • Enterprise workflows are inherently multi-day: procurement, compliance audits, incident investigations
  • "Persistence with guardrails" is the standard — anything less is a prolonged demo
  • Human-in-the-loop is a feature, not a workaround: agents should know when to pause, not just when they're stuck
Bet 4 · Platforms, Not Custom Infrastructure Bet #4

Today: Brilliant engineers drain bandwidth on custom memory systems, bespoke eval harnesses, homegrown observability, handwritten retry logic, and tracing systems that "almost work." Every team rebuilds the same stack from scratch.

The bet: The unique value is domain reasoning — what your agents know about your business, your data, your customers. Everything underneath should be the platform you build on, not plumbing you build with.

  • Open primitives matter: orchestration frameworks that prevent lock-in
  • The same building blocks on your laptop should run in production without a rewrite
  • Time spent rebuilding common infrastructure is time not spent on differentiating logic

The 5-year horizon

Osmani's central argument is an economic one: every engineering month spent rebuilding identity, context, persistence, or orchestration from scratch is a month not spent on the logic only your organisation can write — the domain reasoning, the business rules, the competitive moat.

Teams that pull ahead won't necessarily have better models or bigger compute budgets. They'll have chosen a foundation that treats agents as a managed workforce rather than a collection of scripts, and invested their engineering hours in problems only they can solve.

The window is roughly 12 months. After that, teams that rebuilt the stack themselves will be maintaining it instead of shipping on top of it.

Key Terms
Governance Debt
The silent accumulation of security, audit, and reliability risk from running agents on infrastructure not designed for them.
Stack Ceiling
The hard upper limit on what an agent can do when its infrastructure is underpowered — no amount of prompting gets past it.
Agent Identity
A machine-verifiable, unforgeable credential unique to a specific agent — not inherited from a human or shared service account.
Universal Context
Platform-level access to all relevant business systems simultaneously — CRM, ERP, data warehouse, ticketing — without custom plumbing per integration.
Persistence Primitives
Platform guarantees that agent state survives disconnects, restarts, and credential rotations — enabling multi-day or multi-week workflows.
Human-in-the-Loop
First-class agent behaviour to pause, request permission, and await approval — rather than silently deciding authority or failing.
Undifferentiated Plumbing
Infrastructure every team rebuilds from scratch (retry logic, memory, observability) that provides no competitive advantage.
Session-Bound Execution
The failure mode where agent work is lost when a session token expires, a connection drops, or a model version changes.
