SkillOpt: Teaching AI Agents to Get Better at Their Jobs

What if you could train an AI agent's instruction manual the same way you train a neural network's weights — with gradients, learning rates, and validation?

TL;DR — SkillOpt treats an AI agent's "skill document" (a short set of written instructions) as something trainable. A separate optimizer model watches the agent attempt tasks, proposes small edits to the instructions, and only keeps edits that actually improve performance on held-out tests. The result: +23 points average accuracy from just 1–4 accepted text edits.

The intuition

Imagine you're coaching a new employee. You hand them a one-page playbook: "Here's how to handle spreadsheet tasks." They try a few tasks, and some go wrong. You watch their mistakes, scribble a couple of corrections on the playbook, and have them try again on different tasks to make sure your edits actually helped.

That's SkillOpt. The "employee" is a frozen AI model (it doesn't change). The "playbook" is a short markdown document called a skill. And the "coach" is a separate optimizer model that proposes small, bounded text edits — then checks whether those edits genuinely improved things before committing them.

The key insight: you can apply the discipline of deep learning — learning rates, validation splits, momentum — to optimizing text, not just numbers.

Deep learning training

• Training data → batches
• Learning rate limits weight updates
• Validation set prevents overfitting
• Momentum stabilizes training

SkillOpt

How it works, step by step

1. The agent tries tasks with its current skill

A "skill" is just a markdown document — typically 300 to 2,000 tokens — that gets prepended to the AI model's prompt (or dropped into its workspace as a file). The model itself is frozen: its weights never change. Only the skill document changes.

The agent runs a batch of tasks (say, 40 spreadsheet problems). Each attempt produces a trajectory (the sequence of actions the agent took) and a score (0 or 1 — did it get the right answer?).

2. The optimizer reflects on what went wrong

A separate, typically stronger model (the "optimizer") looks at small groups of failures and successes. It asks: "What patterns do the failures share? What are the successes doing right?" This is the text-space equivalent of computing a gradient — it tells you which direction to push the skill.

3. Edits are bounded by a "textual learning rate"

Here's where SkillOpt gets disciplined. The optimizer can't rewrite the whole skill at once. It's limited to a small number of edits per step (the default is 4), using three operations: add a rule, replace a rule, or delete a rule. This budget decays over time (like a learning rate schedule in deep learning), so early epochs make bigger changes and later epochs fine-tune.

4

The textual learning rate controls how many edits are allowed per step. Cosine decay (default) makes large changes early and fine-tunes later — just like learning rate schedules in neural network training.

4. The validation gate decides

Every proposed edit gets tested on a held-out selection split — tasks the optimizer hasn't seen. If the edited skill scores strictly higher than the current skill, the edit is accepted. If it ties or loses, it's rejected. This is the text-space equivalent of early stopping or validation loss — it prevents overfitting to the training examples.

5. Rejected edits become negative feedback

When an edit is rejected, it doesn't just disappear. It goes into a rejected-edit buffer — a list of "things we tried that didn't work." This buffer is shown to the optimizer in future steps, so it doesn't keep proposing the same failed ideas. Think of it like a to-don't list for the coach.

6. Epochs end with a slow/meta update

At the end of each epoch (a full pass through the training data), the optimizer zooms out. It compares the skill at the start of the epoch to the skill now, identifies what improved and what regressed, and writes protected guidance into a fenced region of the skill. Step-level edits can't overwrite this region — it acts like momentum in deep learning, preserving the gains across training.

Why the discipline matters

Earlier systems for improving agent skills — like having the model rewrite its own instructions after each failure — suffer from unbounded drift. Without a learning rate, each edit can be arbitrarily large. Without validation gating, you can't tell if an edit actually helped. Without a rejected-edit buffer, the system repeats the same mistakes.

BOUNDED VS UNBOUNDED EDITING
UNBOUNDED (no learning rate, no gate)
SKILLOPT (bounded + gated)

Unbounded editing often improves, then crashes when a bad rewrite undoes previous gains. SkillOpt's validation gate ensures the score only goes up (or stays flat when edits are rejected).

SkillOpt was tested across 6 benchmarks, 7 models, and 3 execution environments — 52 combinations total. It won or tied in every single one.

The most striking part: these gains come from just 1 to 4 accepted text edits on a document that's 300–2,000 tokens long. The agent model's weights never change. All the improvement lives in a tiny text file.

Smaller models benefit the most

SkillOpt gave the biggest boosts to the smallest models. The tiny GPT-5.4-nano gained +26.7 points on average — nearly doubling its useful accuracy. This makes sense: smaller models have more room to improve from better instructions, just like a less experienced employee benefits more from a well-written playbook.

Skills transfer across models and tools

A skill optimized for one model can be dropped into a completely different model — or even a different execution environment — and still help. A spreadsheet skill trained inside Codex transferred to Claude Code and gained +59.7 points. The learned rules (like "always inspect the workbook structure before writing formulas") encode general procedures, not tool-specific tricks.

What the learned skills look like

The final skill documents are surprisingly short and readable. Here are some real rules SkillOpt discovered:

SpreadsheetBench: "Inspect workbook structure and formulas before writing. Output evaluated static values, not formula references."
SearchQA: "Infer the expected answer type. Choose the shortest canonical entity that matches."
ALFWorld: "Maintain a visited/frontier ledger. Diversify search strategy after repeated failures."
DocVQA: "Bind each question to the visual row and header. Copy the aligned span exactly."

These are the kinds of rules a human expert might write — but SkillOpt found them automatically through systematic trial and error.

Why this matters

Most AI improvement today comes from making models bigger or training them on more data — both expensive. SkillOpt shows that you can get massive gains by optimizing the instructions instead, after the model is already trained. The training cost is paid once, offline, and the resulting skill file is tiny and portable.

This opens a path where organizations could maintain skill libraries — collections of optimized instruction sets for different tasks — the same way they maintain code libraries today. When a new model comes out, many of the skills would transfer directly.

Key Terms
Skill
A short markdown document (300–2,000 tokens) containing instructions for an AI agent. Gets prepended to the prompt or dropped in the workspace.
Frozen model
An AI model whose internal weights are locked — it can't learn or change. Only the external skill document changes.
Optimizer model
A separate AI model (usually stronger) that reads the agent's attempts and proposes edits to the skill.
Rollout
One complete attempt by the agent to solve a task. Produces a trajectory (action sequence) and a score.
Textual learning rate
The maximum number of text edits allowed per optimization step. Prevents the optimizer from rewriting everything at once.
Validation gate
A check on held-out tasks that only accepts edits which strictly improve the score. The text-space equivalent of validation loss.
Rejected-edit buffer
A running list of edit proposals that failed validation. Shown to the optimizer so it doesn't repeat failed ideas.
Slow/meta update
An end-of-epoch review that writes protected guidance into the skill. Acts like momentum in neural network training.
Epoch
One full pass through all training tasks. SkillOpt typically runs 4 epochs.
Trajectory
The sequence of actions and tool calls an agent makes while attempting a task.
