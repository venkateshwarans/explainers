Gradient
Descent

The algorithm behind almost every neural network — and the one idea that unlocks the rest of ML.

TL;DR — Given a function you want to minimize (your model's prediction error), gradient descent repeatedly asks: which direction is steepest downhill from here? Then takes a small step that way. Repeat until you hit a valley. The critical tuning knob is how big each step should be.

The intuition

Imagine you're blindfolded on a hilly landscape and want to reach the lowest valley. You can't see the terrain, but you can feel which way the ground slopes under your feet. So you step in the steepest downhill direction, feel the slope again, step again. That's gradient descent — navigating a high-dimensional space using only local slope.

In ML, the "landscape" is your model's loss function — a scalar measuring prediction error. Each axis of the landscape is a model parameter (a weight or bias). You want the bottom of the valley: the weights that minimize error.

How it works

For a loss L(θ), the gradient ∇L is a vector pointing in the steepest-uphill direction. Gradient descent steps opposite to it:

θt+1 = θt − η · ∇L(θt)

where η (eta) is the learning rate. For the demo above, ∇f = (2x, 16y) — a simple linear gradient. The y-axis has 8× stronger curvature, which is why it zigzags in y and converges smoothly in x. In real networks, the loss landscape is billions of dimensions deep and far noisier, but the same sensitivity to η dominates.

Momentum — smoothing out the zigzag

Vanilla GD bounces in narrow valleys because it responds only to the current slope. Momentum adds a velocity term — the optimizer builds up speed in consistent directions and resists reversals:

vt = β · vt−1 + (1−β) · ∇L(θt)
θt+1 = θt − η · vt

Try momentum ≥ 0.7 in the demo above — the zigzag collapses into a smooth arc, and you can use a higher learning rate without diverging.

Where you'll meet it

Every LLM you've used was trained by Adam running gradient descent for trillions of steps across billions of parameters. The loss function measures how surprised the model is by the next token. Each step nudges the weights fractionally toward better predictions. After enough steps, those weights encode a working model of language.

Outside ML: energy minimization in physics simulations, utility optimization in economics, topology optimization in structural engineering — any continuous parameter search where you can compute a gradient.

Key Terms
Loss function
Scalar measuring prediction error. Gradient descent minimizes it.
Gradient ∇L
Vector of partial derivatives — points uphill. GD steps opposite to it.
Learning rate η
Step size. Too large → diverge. Too small → slow. The most critical hyperparameter.
Parameters
Numbers the model learns — neural network weights and biases.
Momentum β
Exponential average of past gradients. Smooths updates; lets the optimizer gather speed.
Convergence
When loss stops decreasing — the optimizer found a (local) minimum.
Adam
Momentum + adaptive per-param learning rates. Default optimizer for most deep learning today.
Saddle point
∇L = 0 but not a minimum: downhill in some directions, uphill in others.
